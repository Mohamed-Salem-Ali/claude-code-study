## The Edit → Review → Accept Flow

1. Describe what you want in plain English.
2. Claude shows a **diff** of proposed changes.
3. You **accept** (Enter) or **reject** (Esc / n).
4. Claude runs follow-up checks (tests, linting) if needed.

## Tips for Good Change Prompts

- Be specific about *what* and *where*: "In `src/lib/auth.ts`, add refresh token support."
- Give acceptance criteria: "The test suite should still pass."
- Reference existing patterns: "Follow the same pattern as `create-project.ts`."
